RoboJournal ChangeLog

Version 0.4.2

Release Date: 9/13/13

The following changes were made in this release:

  1. Split documentation into a separate robojournal-doc package on Debian-based Linux (including Ubuntu, Linux Mint, et al) because including documentation in each architecture-specific package is redundant when the documentation itself is platform-agnostic.
    1. Fixed typos and minor corrections to documentation (9/2/13).
  2. Added "pkg" folder for Linux package mangement files.
    1. Removed Fedora build patch since it is no longer needed (6/1/13).
    2. Added SPEC file for creating Fedora ≥18 packages.
    3. Added control file and everything else required to build Debian/Ubuntu packages (9/2/13).
  3. Temporarily disabled spellcheck on Linux-based systems to prevent a common crashing bug from occurring.
  4. Removed Fedora patch question/logic from Linux compile script (6/1/13).
  5. Removed caps-lock detection code from the DBLogin class (6/1/13).
  6. Bugfix: Fixed "ugly 16x16 icon" bug on Windows 7. This bug caused an overstretched icon to be used on the ALT+TAB window list and similar places. Backported from RoboJournal 0.5 source (9/11/13).
  7. Bugfix: RoboJournal no longer displays redundant error messages if the documentation files are missing or installed incorrectly (9/11/13).
  8. Bugfix: Replaced the documentation build script with a far more robust Perl-based solution. If the documentation fails to build correctly on the first attempt, the new build script tries several different executables until it finds one that works. This fix is necessary because Qmake often tries to use Qt5 tools on systems that have both Qt4 and Qt5 installed.
  9. Bugfix: The vertical splitter bar on the MainWindow now remembers its position between program sessions. The splitter only reverts to its default 50/50 position if the saved position data is unavailable.
  10. Added description/explanation of new features to RoboJournal Documentation.
    1. Bugfix: Documentation version numbers are no longer hardcoded in the help project file and collection file.

Version 0.4.1

Release Date: 4/5/13

The following changes were made in this release:

  1. The Windows version of RoboJournal is now compiled against Qt 4.8.4 instead of Qt 4.7.4 (which was used in versions 0.1–0.4). As such, most of the dynamic link libraries (DLL files) shipped with the Windows version of RoboJournal are brand-new.The purpose of the Qt version increment is to keep pace with the version of Qt in the Debian repositories.
  2. The current version number is no longer listed in the title bar on RoboJournal's main window.
  3. The tooltip for the Entries tab on the main window now reads, "Display the Chronological Entry List" in accordance with the official name of the object in the new documentation.
  4. Changed the icon for the Clear button on the Appearance Page in RoboJournal Preferences.
  5. Changed the layout of the About RoboJournal dialog.
  6. Bugfix: The Font Color field on the Appearance Page in RoboJournal Preferences no longer accepts direct input (necessary to prevent the user from entering an invalid color value).
  7. Renamed the Select default Journal drop-down menu entry to Journal Selector.
    1. Changed the Select default journal window title to "Journal Selector".
  8. Bugfix: Corrected spacing issue affecting Linux Mint (and other distros with larger-than-average text) in the Welcome to RoboJournal window displayed during First-Run.
  9. Bugfix: The current search term now requires the Enter key to be pressed only once in order to run a search; in 0.4 the user had to press the key twice to trigger the Search button.
  10. Bugfix: The data from the most recent search is now automatically cleared whenever you change the search type (any type of pattern match to tag search and vice versa).
  11. Bugfix: Clicking the Search for whole words button now causes focus to return to the search term box automatically (thereby saving the user an extra click).
  12. Added "Speech Balloon" icon option to entry node choices.
  13. Added ALT key shortcut combos to all boolean (selectable/unselectable) options in RoboJournal Preferences. The underlined letter in the item description indicates which key is required to trigger the option (i.e. "Option 1" would be triggered by ALT+O).
  14. Added Tag Reminder system. This feature displays a window that notifies you of any untagged entries in your journal each time you log in. The Tag Reminder is designed to get the users into the habit of tagging their entries.
    1. Added option to enable/disable the Tag Reminder window to the "General" Page in RoboJournal Preferences.
  15. Bugfix: Eliminated the possibility of the user declaring a "null" tag by using a more sophisticated Regular Expression to detect attempts instead of relying on a simple == operator.
  16. Added new feature that allows the name of the current journal to be displayed in the main window title bar.
    1. Added an option to enable/disable this behavior to the "General" Page in RoboJournal Preferences.
  17. Bugfix: Disconnecting while there are search results present on the Search tab no longer crashes the program.
  18. Added comprehensive documentation for RoboJournal 0.4.1 (displayed with the included Qt Assistant program).

Version 0.4

Release Date: 12/17/12

The following changes were made in this release:

  1. Bugfix: The background image settings are now properly disabled when the user enables the custom style setting. These settings only become active if the user selects a background image.
  2. Bugfix: The Use default host and port checkbox on the Journal Selector window now obeys the current default values setting in the configuration (with one exception: it is always enabled during first-run configuration).
  3. Improved Entry Exporter to allow the user to specify his or her choice in font/font size for headers and body text in exported HTML documents. This feature is configured through the new Export Settings page in the Preferences window.
  4. Modified date settings to allow the user to show the day of week when an entry was made.
  5. Added batch file to simplify the removal/cleanup of compiled executables, object code, and compiler-generated headers between builds. FYI: This tool is only available in the source package because its purpose is to revert the source directory to its original state after a compile operation has taken place.
  6. Added the ability to use the same font settings in both the Editor and Entry Output pane in the main window. This gives the user a better idea of how the finished entry will look once it has been saved.
  7. Added automated post-processing features to Editor:
    1. Added whitespace removal feature which cleans unnecessary line breaks and redundant spaces between words from the body text.
    2. Added HTML-compliant dashes and hyphenation; all instances of -- are replaced with — and dashes between numbers are properly elongated (i.e. 1–2).
    3. Added smart quotes feature which replaces straight (") quotes with curvy (“) quotes. This feature also affects apostrophes and other single quotes in words.
    4. Added miscellaneous processing features which convert sequences of ≥ 3 adjacent periods to the proper ellipsis character (…) and formalizes number notation (i.e. 1st, 112th).
  8. Added resistance to common forms of SQL injection attacks. FYI: Although this new feature provides some degree of protection, users should not assume they are 100% safe or immune to these attacks.
  9. Bugfix: Solved layout issues which sometimes caused parts of the user interface to extend beyond the edges of its window or container object (due to large font size usage at the operating system level). The user interface for each form now properly accommodates all but the largest font sizes without issue.
  10. Bugfix: RoboJournal now requests confirmation if the user quits the program while the Search tab is active.
  11. Added spellcheck functionality (based on Hunspell) to the Editor. RoboJournal comes with a standard US English dictionary but other dialects/languages are also supported. Other dictionaries must be downloaded from third-party sources (compatible dictionaries consist of a DIC/AFF file pair).
  12. Added new icons to the program and changed many existing ones.
  13. Changed wording on Entry Exporter dialog tabs.
  14. Resizable windows now remember their previous size and position between sessions.
  15. Converted the Editor into a toolbar-based layout.
  16. Main Window, Editor and Preferences windows are now resizable and remember their sizes between sessions.
  17. Bugfix: (Ubuntu-specific) The menu bar in the Main Window remains attached to the window instead of inheriting the standard Unity menu bar behavior.
  18. Bugfix: Icon-only toolbar button setting (if enabled) is now preserved after the user manually launches the Journal Creator or Journal Selector.
  19. Bugfix: Any active journal connections now automatically terminate if the user manually launches the Journal Creator or Journal Selector.
  20. Added Newconfig class. This is meant to be a full replacement (and an extension) of the original Preferences window. The Newconfig class features a highly modular design with potential for future expansion.
    1. Added ConfigurationGeneral, ConfigurationJournal, ConfigurationEditor, ConfigurationExport, ConfigurationAppearance, and ConfigurationMySQL classes. Each of these is a separate module (represented as a page on the Preferences window) which controls specific settings and interacts with the Newconfig class.
    2. Users now have the ability to override the colors of the date/timestamp box and tag box when using a custom theme. It is now possible to use system colors and a custom theme simultaneously if desired. In prior releases, the background of those elements were always the same color as the body text; this new function is ideal for those who wish to use a custom font but retain the native OS background.
    3. Deprecated: Old Config class (config.cpp, config.h, config.ui). Even though the original Preferences interface worked fine at first, RoboJournal has grown in size and complexity over the last few releases to the point where the old interface had become overcrowded and further expansion was impractical.
    4. Removed deprecated Config class files from source package and all references to them in the source code.
    5. Changed Sort by Day option text to Group journal entries by day.
    6. Added Display full name (instead of username) option. This displays the user's full name instead of the abbreviated username in the entry header.
    7. Added post-processing options to the Editor page.
    8. Added Export Preview function to the Export page.
    9. Added Highlighter-related options to the Journal page.
  21. Win32 executables generated through the Build Helper Script are now properly stripped (MinGW strip utility). Stripping removes unneeded data from the executable which improves efficiency and reduces file size.
  22. Added menu shortcut launcher for Linux systems (robojournal.desktop) to source package (launcher will appear in desktop environment/window manager apps menu after the user runs "make install"). This also (hopefully) resolves a bug affecting versions 0.1 - 0.3 where the RoboJournal Debian package sometimes fails to create a menu entry after installation.
  23. Changed Journal Creator launcher text in the Main Window to say "Journal Creator" instead of "Create a New Journal".
  24. Added SettingsManager class:
    1. All settings are now handled through SettingsManager class.
    2. Changed "STAGE 1" startup text to reference the new SettingsManager Class.
    3. Deprecated: Original configuration file ($HomePath/.robojournal/robojournal.cfg). Configuration settings are now stored in $HomePath/.robojournal/robojournal.ini instead.
    4. Deprecated: ConfigManager class. Removed configmanager.cpp and configmanager.h (and all references to them in the source code) from the source package.
  25. Added shortcut key (Q) to the exit confirmation dialog in Main Window.
  26. Added gender check during first-run configuration which sets a different username avatar icon for female users.
  27. Removed extra space around the edge of the layout in the Main Window.
  28. Added search feature to main window:
    1. Added tabbed interface to main window splitter.
    2. Placed journal entry tree in the Entries tab.

Version 0.3

Release Date: 8/28/2012

The following changes were made in this release:

  1. Changed RoboJournal logo image in FirstRun and AboutRJ classes.
  2. Fixed bugs in JournalCreator class:
    1. Bugfix: System username is now displayed on Linux/Unix systems.
    2. Bugfix: Added more padding below the "Passwords [do not] match" text. This keeps the text from being too close to the bottom of the frame.
    3. Added tooltips to the Journal Name, Journal Host, and Username fields.
  3. Simplified and changed text on FirstRun class UI file. This solves a bug where the intro paragraph text size changed depending on system font size.
  4. Added Build Helper Script (for Windows only).
  5. Bugfix: Fixed a bug that prevented the current color in use from being auto-selected in the Config class color chooser dialog.
  6. Bugfix: Fixed incorrect error reporting when journal creation process fails due to a missing Qt MySQL driver.
  7. Bugfix: Added an error message that is triggered if you try to create a journal where journal@host already exists.
  8. Bugfix: Changed project file to ensure that builds are compiled in release mode by default on Windows systems.
  9. Added a shortcut/menu icon (F11) to expand/collapse entry tree.
  10. Added a check for Caps Lock to DBLogin class.
  11. Changed exit warnings in Editor class.
  12. Bugfix: Removed extra padding on MainWindow::Toolbar when buttons are set to image-only.
  13. Modified DBLogin Class:
    1. Removed root login warning label and replaced it with QMessageBox warnings/errors. This helps reduce crowding/sizing problems on large-text systems.
    2. TextBoxes are set to QLineEdit::ReadOnly instead of QLineEdit::SetDisabled when UseDefaults==true.
  14. Changed text for error (critical) QMessageBoxes in Editor class.
  15. Fixed program icon for Linux systems (previous versions used the too-small Windows icon that looked bad when stretched).
  16. Added journal export system to export database entries (or the whole database) to HTML or plain text:
    1. Added Save button to toolbar.
    2. Added Save entry and CRTL+S shortcut to File menu.
  17. Removed expand/collapse widget on root node (journal name) in entry tree.
  18. Changed MainWindow::Statusbar text:
    1. Reworded the welcome message (Click Connect (or press F2) to begin working with a journal database.) and the New Journal message (Welcome to your new journal! Click Write in Journal (or press F4) to get started.).
    2. Separated Username text into its own pane.
    3. Added images to Username and Entry Count panes.
  19. Config class changes:
    1. Enlarged Config window to reduce crowding in some GNOME setups.
    2. Added Apply button to Config::QDialogButtonBox.
    3. Made auto-load feature a config menu option. This lets the user decide if he/she wants the most recent entry to load automatically after login.
  20. Added tagging system (Tagger class):
    1. Modified MySQLCore::Update() to prevent NULL data from being written to the tags column in database entries table.
  21. Added JournalSelector class:
    1. FirstRun class now calls JournalSelector if user decides to use existing journal during config.
    2. Added a private function that executes a SHOW DATABASES tranaction to MySQLCore class.
  22. Changed window icon to database_add.png in JournalCreator class.
  23. Rewrote changelog.html from scratch to be XHTML 1.0 compliant.
  24. Replaced original compile/installation documentation (INSTALL-GUIDE-r3.htm) with an XHTML 1.0 compliant document (compile-instructions.xhtml).

Version 0.2.1

Release Date: 5/11/2012

The following changes were made in this release:

  1. Added "make install" and "make uninstall" support for Linux/OS X (RoboJournal gets installed to /usr/local/bin).
  2. Added Linux/OS X installation instructions to INSTALL-GUIDE-r3.htm.
  3. Reformatted Changelog.htm to pass HTML 4.01/STRICT.
  4. Removed MySQL header dependencies in source to ensure full GPL3 compliance.
